# VLM 화재·재난 현장 해석 평가 🔥

엣지 디바이스(YOLO)가 캡처한 화재·재난 현장 장면을, 비전언어모델 **Qwen3-VL-2B-Instruct**가 한국어로 얼마나 정확·상세히 해석하는지 평가하고 프롬프트·입력 파라미터를 튜닝한 프로젝트입니다.

**핵심 성과 — 위험 유형(화재 / 낙상 / 전도 / 없음) 종합 분류 정확도 71% → 89%**

## 핵심 성과 — 왜 이 산출물들이 나왔나

모델이 헷갈리던 장면(전도, 연기 등)을 6단계로 하나씩 바로잡아 종합 유형 정확도를 **71% → 89%**로 끌어올렸습니다.

| 단계 | 문제 | 해결 | 효과 |
|---|---|---|---|
| 1. 모델 선정 | Qwen2-VL-2B가 한국어 지시를 거부하고 영어로만 동작 | 한국어 네이티브 Qwen3-VL-2B 채택 | 한국어 해석 정상 동작 |
| 2. 프롬프트 개선 (P1→P5) | "가능한 한 자세히" 지시가 장황함 유발 | 해당 지시 제거 | 평균 길이 111자로 단축, 정확도 유지(이미지 94% / 영상 83%) |
| 3. 입력 파라미터 | 영상 프레임 수 부족으로 위험 인식 실패 | 8→16프레임 + 해상도 560px | 기계 전도 판정 60% → 100% |
| 4. 유형 어휘 분기 | '전도'를 모델이 전부 '낙상'으로 오인 | `낙상(사람)` / `낙상(기계)`로 주체 분리 | 전도 유형 0% → 100% |
| 5. 연기 회귀 보정 | '낙상' 옵션 2회 등장으로 연기 장면이 낙상으로 끌림 | "연기·불꽃이 보이면 화재 우선" 규칙 추가 | 연기 회복, 종합 유형 71% → 89% |
| 6. 한국어 강제 | 한국어 지시에도 중국어 한자 혼입 | 디코딩 단계에서 한자 토큰 확률 차단 (LogitsProcessor) | 한자 39자·6건 → 0자 |

**최종 카테고리별 정확도** (판정 / 유형)

| 카테고리 | 이미지 | 영상 |
|---|---|---|
| 화재 | 100 / 100 | 100 / 100 |
| 연기 | 86 / 57 | 100 / 100 |
| 낙상 | 100 / 100 | 60 / 100 |
| 전도 | 100 / 100 | 80 / 100 |
| 정상(주차/작업) | 80~100대 | 80~100대 |

→ **종합 유형 정확도: 이미지 82% · 영상 94%**

## 데이터 · 실행 · 코드

### 데이터

이미지 34장 + 영상 36개. 위험(화재·연기·낙상·전도)과 정상(주차장·작업장) 장면. 정답은 파일 접두사로 식별합니다.

| 접두사 | 정답 유형 |
|---|---|
| `fire`, `smoke` | 화재 |
| `person_fall` | 낙상 |
| `machine_tipover` | 전도 |
| `normal_parking`, `normal_worker` | 없음(정상) |

> 데이터셋은 용량이 커서 저장소에서 빠질 수 있습니다. 현장 YOLO 캡처를 같은 접두사 규칙으로 `dataset`에 넣고 돌리면 됩니다.

### 실행

```powershell
# 추론 평가 (GPU) — 모델: Qwen3-VL-2B (fp16, RTX 4060 8GB)
venv\Scripts\python.exe run_benchmark.py --media image,video

# 결과 md → 이미지 임베드 PDF
venv\Scripts\python.exe tools\report_to_pdf.py
```

### 코드 구조

**코어** (vlm 루트)

| 파일 | 역할 |
|---|---|
| `config.py` | 모델·카테고리·프롬프트·생성/프레임 파라미터 |
| `vlm_engine.py` | 모델 로드·추론 + 한자 차단(FORCE_KOREAN) |
| `run_benchmark.py` | 추론 후 판정/유형 정확도·지연 집계 → 리포트(md+json) 저장 |

**보조 도구** (`tools/`)

| 파일 | 역할 |
|---|---|
| `rebuild_report.py` | json 캐시로 md 재생성 (GPU 불필요) |
| `report_to_pdf.py` | md를 이미지 임베드 PDF로 변환 |
| `param_sweep.py` / `analyze_sweep.py` | 입력 파라미터 스윕·분석 |
| `experiments.py` | 검증 실험 모음 (term / smoke / repeat / korean / all) |
